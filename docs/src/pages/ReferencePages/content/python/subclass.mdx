# Python Subclassing Guide

LCAx Python classes are implemented in Rust using PyO3 and support subclassing. This allows you to extend LCAx classes with custom methods, attributes, and inheritance hierarchies while maintaining full compatibility with the original functionality.

## Overview

All Python classes exported from LCAx can be subclassed. This includes:

- **Core Classes**: `EPD`, `Product`, `Assembly`, `Project`
- **Impact Classes**: `ImpactCategory`, `Impacts`, `GenericData`
- **Validation Classes**: `ValidationSchema`, `ValidationRule`, `ValidationResult`
- **Project Classes**: `BuildingInfo`, `Location`, `SoftwareInfo`
- **Classification Classes**: `Classification`, `Transport`
- **Utility Classes**: `AreaType`, `ValueUnit`

### Maintaining Compatibility

When subclassing LCAx classes:

1. **Preserve original constructors**: Always call `super().__init__()` with required parameters
2. **Maintain type signatures**: Keep method signatures compatible with parent classes
3. **Use proper inheritance**: Ensure your subclass inherits from the correct LCAx class
4. **Test thoroughly**: Verify that both original and custom functionality works correctly

### Performance Considerations

- Subclassing adds minimal overhead to LCAx classes
- Custom methods are executed in Python, while original methods remain in Rust for performance
- Memory usage is slightly increased for custom attributes
